Django Framework

Einfache Anleitung zum Einbinden des Frontend-Codes in Django

1. Verwendung des Master-Templates

  1. Erstelle ein HTML-Template für das Master-Template, z.B. "master.html".
  2. Definiere in diesem Template den HTML-Code für den Header, Footer und andere gemeinsame Elemente der Benutzeroberfläche.
  3. Verwende den Django Template Language-Syntax { % block content % }{ % endblock % }, um einen Bereich im Master-Template zu markieren, in den der spezifische Inhalt eingefügt werden kann.
  4. Speichere das Master-Template im Verzeichnis "templates" deiner Django-Anwendung.
  5. Erstelle für jede Unterseite deiner Anwendung ein spezifisches Template und erweitere es vom Master-Template. Verwende dazu den Django Template Language-Syntax { % extends 'master.html' % } am Anfang des Templates.
  6. Verwende den Django Template Language-Syntax { % block content % }Hier kommt der spezifische Inhalt der Unterseite{ % endblock % } im spezifischen Template, um den Inhalt in den entsprechenden Bereich des Master-Templates einzufügen.
  7. Speichere die spezifischen Templates ebenfalls im Verzeichnis "templates" deiner Django-Anwendung.

2. Einbinden statischer Dateien

  1. Erstelle einen Ordner namens "static" im Hauptverzeichnis deiner Django-Anwendung.
  2. Innerhalb des "static"-Ordners kannst du weitere Unterordner für verschiedene Arten von statischen Dateien erstellen, z.B. "css" für CSS-Dateien oder "js" für JavaScript-Dateien.
  3. Kopiere die entsprechenden CSS- und JavaScript-Dateien in die entsprechenden Unterordner im "static"-Ordner.
  4. In deinen Django-HTML-Templates kannst du den Django Template Language-Syntax { % load static % } verwenden, um die Verwendung von statischen Dateien zu aktivieren.
  5. Verwende den Django Template Language-Syntax { % static 'pfad/zur/datei.css' % } oder {% static 'pfad/zur/datei.js' %} in den entsprechenden HTML-Templates, um die statischen Dateien einzubinden. Passe den Pfad entsprechend dem tatsächlichen Pfad zu den statischen Dateien an.

3. Verlinken von Unterseiten in der Sub-App

  1. In deinem spezifischen Template kannst du den Django Template Language-Syntax { % url 'app_name:url_name' % } verwenden, um auf eine spezifische Unterseite in der Sub-App zu verlinken. Ersetze "app_name" durch den Namen der Sub-App und "url_name" durch den Namen der URL-Konfiguration für die Unterseite.

Durch die Verwendung des Master-Templates, das Einbinden statischer Dateien und das Verlinken von Unterseiten in der Sub-App können deine Schüler ihren Frontend-Code effektiv in ihre Django-Anwendung integrieren und eine konsistente Benutzeroberfläche mit wiederverwendbaren Komponenten erstellen.
